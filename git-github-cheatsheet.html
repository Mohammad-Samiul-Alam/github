Git & GitHub — Field Notes

A working reference for branches, identities, and shipping code to GitHub.
CONFIG

Email identity

Check which email Git is signing commits with, then change it per-repo or globally.
Check current email
1git config user.email
2git config --global user.email
Set a new email
3git config user.email "your-new-email@example.com"
4git config --global user.email "your-new-email@example.com"
BRANCH

Switch to a new feature branch

1git checkout -b feature
UPLOAD

Push a project to GitHub for the first time

Option A — manual repo creation via the CLI prompt:
1gh auth login
2git init
3git add .
4git commit -m "first commit"
5git config user.email "your-new-email@example.com"
6git branch # check current branch
7gh repo new
OR
Option B — one command creates and links the remote:
1gh auth login
2git init
3git add .
4git commit -m "first commit"
5git branch # optional check
6gh repo create <repository-name> --public --source=. --remote=origin
7# shorthand: gh repo create my-awesome-project -p -s . -r origin
8# or link manually: git remote add origin https://github.com/your-username/repo-name.git
9git push -u origin main
UPDATE

Push changes to an existing repository

Step 1 — connect to the repo (new repo vs. linking an old one):
1gh auth login
For a new repository
2git init
3git add .
For an existing repository
4git remote add origin https://github.com/yourusername/your-repo.git
Step 2 — commit and push. Pick the path that matches your intent:
⚠ FORCE

Overwrite remote history (deletes anything not in your local branch)

1git add .
2git commit -m "Merge conflict resolved"
3git push --force origin main
✓ SAFE

Only update, keep everyone's history intact

1git pull origin main --rebase
2git push -u origin main
3git add .
4git commit -m "Merge conflict resolved"
5git push -u origin main
